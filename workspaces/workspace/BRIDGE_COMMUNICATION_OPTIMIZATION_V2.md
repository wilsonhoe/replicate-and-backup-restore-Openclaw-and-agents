## ENHANCED COORDINATION PROTOCOL

### 1. EXECUTE-FIRST COMMUNICATION RULE

**New Standard:**
```
IF command_given AND prerequisites_met:
    EXECUTE_IMMEDIATELY
    REPORT_RESULTS_ONLY
ELSE:
    REPORT_BLOCKER_BRIEFLY
```

**Application:**
- Wilson says "use Zoho" → Execute Zoho, report "Posted Day 3"
- Claude says "implement template" → Implement template, report "Template active"
- No more "ready to" or "waiting for" when prerequisites are met

### 2. BRIDGE STATE SYNC PROTOCOL

**Before Every Response:**
1. Check bridge files for new decisions
2. Update internal operational state
3. Execute any pending commands
4. Only then respond with results

**Decision Capture Template:**
```markdown
**DECISION CAPTURED [timestamp]**
- Source: [Claude/Wilson/Bridge]
- Decision: [What was decided]
- Action Taken: [What was executed]
- Result: [Outcome with proof]
```

### 4. MEMORY-TO-EXECUTION PIPELINE

**Problem:** Treating memory as search engine instead of operational state

**Solution:**
1. **Memory Check** → Find decisions/context
2. **State Update** → Apply to current operation
3. **Execution** → Act based on updated state
4. **Confirmation** → Report results only

**Anti-Pattern to Eliminate:**
- ❌ Search memory → Report findings → Ask for confirmation → Wait → Repeat

**Pattern to Implement:**
- ✅ Search memory → Update state → Execute → Report completion
