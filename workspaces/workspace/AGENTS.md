## Agent Verification Protocols

### System State Verification
**BEFORE claiming a tool is broken or unavailable:**
1. Check if another agent or process has already proven it works
2. Test current state rather than relying on outdated assumptions
3. Acknowledge when functionality is demonstrated by others
4. Update understanding immediately when proven wrong

**Example:** Claiming "exec denied: allowlist miss" when Claude had already successfully executed scripts multiple times. The real issue was Twitter's React editor, not exec permissions.

### Error Investigation Priority
**WHEN troubleshooting failures:**
1. Verify the actual error message and context
2. Distinguish between permission issues vs application logic issues
3. Check if the failure is in a specific component (e.g., Twitter React vs exec tool)
4. Focus on the root cause, not assumed blockers

**Pattern:** Exec tool working → Twitter React editor disabled → Fix input method, not execution permissions
